# Figures — cross-vendor heterogeneity-guard paper

Each figure is generated by a self-contained Python script. The scripts emit both PDF (for LaTeX inclusion at submission time) and PNG (300 dpi, for markdown preview and supplementary materials).

## How to rebuild

```bash
cd papers/cross-vendor-heterogeneity-guard/figures
python3 figure1_architecture.py
python3 figure2_catches.py
```

Dependencies: `matplotlib` (already installed user-local at `~/Library/Python/3.13/` per `requirements.txt` of the Ar9av/PaperOrchestra workspace).

## Figures

### `figure1_architecture.py` → `figure1_architecture.{pdf,png}`

Cross-vendor heterogeneity guard architecture. Trigger → Pass 1 (Anthropic) → Pass 2 (DeepSeek/Google) → Pass 3 (Anthropic/OpenAI) → synthesis/queue/, plus an episodic DeepSeek peer-review pass (dashed) feeding an audit-trail surface.

### `figure2_catches.py` → `figure2_catches.{pdf,png}`

Four representative catches surfaced during the 2026-04-25 → 2026-05-08 operational window, color-coded by surfacing vendor. The §5.4 DeepSeek Connection 7 catch is annotated as the seminal cross-vendor surfacing that motivated the architecture itself.
